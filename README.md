# A股行业轮动分析网站（优化版）

这是一个面向 **A 股行业轮动观察** 的可视化网站。项目沿用原版本的**在线数据链路**，通过服务端拉取主要指数与申万行业指数行情，在前端展示行业与指数的归一化走势、相对沪深300的超额收益，以及基于 **RS 动量斜率** 的行业排名轮动结果。[1]

当前版本重点完成了三项优化。其一，页面顶部显示的行业数量已经与下方实际可切换的行业按钮数量统一；其二，主图右侧的行业涨幅列表与下方行业按钮区已重新排布，避免了视觉重叠；其三，在原有走势图下方新增了 **行业排名轮动图**，并支持按不同时间窗切换 RS 动量排名逻辑。

## 核心能力

| 模块 | 说明 |
| --- | --- |
| 行业与指数综合走势 | 展示主要指数与申万一级行业在所选时间窗内的归一化涨跌幅走势。 |
| 最新行业涨幅列表 | 在主图右侧显示当前时间窗下行业的最新涨幅排序，并与图表颜色联动。 |
| 行业按钮切换 | 支持对单个行业进行高亮、隐藏与对照查看，便于聚焦主题板块。 |
| 行业排名轮动图 | 以排名折线展示行业强弱顺位变化，纵轴为 1 至 n 名，且第 1 名位于最上方。 |
| 超额收益图 | 展示行业相对沪深300的超额收益变化，用于观察风格强弱。 |
| 在线刷新 | 支持基于当前时间窗重新请求最新数据。 |

## 行业口径说明

本版本**固定采用 29 个申万一级行业口径**。这是当前项目明确确认后的展示范围，**不补入“社会服务”和“综合”**。因此，页面顶部行业统计、主图可绘制行业数量、右侧排行列表以及下方按钮区，都会统一以这 29 个行业为准。

项目内置的一级行业包括：农林牧渔、基础化工、钢铁、有色金属、电子、汽车、家用电器、食品饮料、纺织服饰、轻工制造、医药生物、公用事业、交通运输、房地产、商贸零售、银行、非银金融、建筑材料、建筑装饰、电力设备、机械设备、国防军工、计算机、传媒、通信、煤炭、石油石化、环保、美容护理。

## 排名轮动算法说明

行业排名轮动图的核心思想，是把行业相对沪深300的强弱变化转成**可连续观察的排名轨迹**。项目先计算每个行业在选定区间内的相对强度序列，即：

> **行业涨幅% − 沪深300涨幅%**

随后再按对应时间窗，计算该相对强度序列的动量斜率，并进行起点归一化处理。每个交易日都根据这一 RS 动量值重新排序，从而形成“行业排名轮动图”。

当前时间窗与 RS 动量窗口的映射如下。

| 页面时间窗 | RS 动量窗口 | 用途说明 |
| --- | --- | --- |
| 7天（7D） | 3日 | 用于捕捉更短周期的情绪变化。 |
| 1个月（1M） | 5日 | 用于识别周级别强势题材。 |
| 3个月（3M） | 20日 | 作为核心观察窗口，识别月度级别的爆发拐点。 |
| 6个月（6M） | 20日 | 用于观察季度主线延续性。 |
| 1年（1Y） | 60日 | 用于过滤短期噪声，确认年度风格轴心。 |

为了改善 29 条排名线同时显示时的可读性，当前界面会优先强调前排行业，并通过较轻的背景线与更清晰的主线层级降低视觉拥挤感。

## 数据来源与实现方式

本项目的数据抓取逻辑主要位于服务端 `server/stockData.ts`。前端页面本身不直接请求外部行情源，而是通过站内接口获取清洗后的统一数据结构。这也是原仓库里“页面代码没有直接体现抓取逻辑”的原因。

数据实现采用以下方式：

| 项目层级 | 位置 | 作用 |
| --- | --- | --- |
| 数据源接入 | `server/stockData.ts` | 维护申万行业与新浪指数代码映射，并拉取行情数据。 |
| 接口层 | `server/routers/stock.ts` | 对外提供前端消费的数据接口与刷新能力。 |
| 分析算法 | `client/src/lib/industryAnalytics.ts` | 统一行业数量口径、超额收益计算与排名轮动算法。 |
| 展示层 | `client/src/pages/Home.tsx` | 负责页面布局、交互与图表呈现。 |

## 本地运行方式

项目基于 **React + Vite + Express + tRPC** 构建，适合继续在当前结构上迭代。

### 1. 安装依赖

```bash
pnpm install
```

### 2. 启动开发环境

```bash
pnpm dev
```

启动后，前端与服务端会在同一个开发入口下运行。

### 3. 运行测试

```bash
pnpm test
```

当前测试已覆盖两类关键逻辑：一类是 **29 个行业口径约束**；另一类是 **行业排名轮动算法的核心计算**。

### 4. 生产构建

```bash
pnpm build
pnpm start
```

## 当前版本变更摘要

相较于较早版本，本次优化主要解决了以下问题。首先，页面顶部行业数量与实际按钮数量已经统一，不再出现统计口径与展示口径不一致的问题。其次，主图右侧“最新行业涨幅”区域已经独立成侧栏，与下方按钮区不再重叠。最后，新增的行业排名轮动图使得行业强弱切换可以被连续观察，而不再只能看某一时点的静态涨幅。

## 使用说明与注意事项

本项目仅作为**市场观察与行业轮动分析辅助工具**，不构成投资建议。由于数据来自在线行情接口，若外部接口临时波动、限流或返回字段变化，个别时间窗下的刷新速度和可用性可能受到影响。[1]

如果后续继续扩展，本项目很适合增加“只看前 5 / 前 10 行业”“点击行业联动个股列表”“排名变化提醒”等功能，以进一步增强实战分析体验。
